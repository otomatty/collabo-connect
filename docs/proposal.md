## 1. エグゼクティブサマリー

Collabo-Connect は、**AIインタビューとグループ募集掲示板**を組み合わせた社内コミュニケーションプラットフォームです。社員一人ひとりの人柄や興味をAIが自然に引き出し、「ごはん・勉強会・イベント」を通じた部門を超えた交流を促進します。

導入により、以下の効果が期待されます：

- **社員間の心理的安全性の向上**（部門横断の交流機会が増加）
- **オンボーディングの加速**（新入社員が既存メンバーと早期につながる）
- **ナレッジシェアの促進**（勉強会・技術相談が自然と発生）
- **離職率の低減**（帰属意識・エンゲージメントの向上）

---

## 2. 背景と課題

### 現状の課題

| # | 課題 | 影響 |
|---|---|---|
| 1 | **部門間の交流不足** | チーム外のメンバーとの接点がなく、コラボレーションが生まれにくい |
| 2 | **新入社員の孤立** | リモートワーク下では雑談の機会が少なく、馴染むまでに時間がかかる |
| 3 | **社員の人柄が見えにくい** | 業務上のやり取りだけでは、趣味や興味を知る機会がない |
| 4 | **交流イベントの企画コスト** | 有志の交流会は告知・参加者管理が個人頼みになり、継続しにくい |
| 5 | **ナレッジの属人化** | 誰が何に詳しいかが分からず、適切な相談相手を見つけにくい |

### 解決の方向性

**「自然な仕掛け」で社員同士の接点を生み出す**プラットフォームを提供し、交流を"頑張るもの"から"自然に起きるもの"に変える。

## 3. ソリューション概要

### Collabo-Connect とは

モバイルファーストで設計された社内コミュニケーションWebアプリです。スマートフォンのブラウザから手軽にアクセスでき、以下の3つの柱で社員交流を支えます。

```
┌──────────────────────────────────────────────────┐
│                Collabo-Connect                    │
├──────────┬──────────────────┬─────────────────────┤
│  AI      │  グループ募集     │  メンバー            │
│  インタ   │  掲示板          │  ディスカバリー       │
│  ビュー   │                  │                     │
├──────────┼──────────────────┼─────────────────────┤
│ 毎日の    │ ごはん・勉強会・  │ タグ・AI紹介文で     │
│ 質問に    │ イベントの        │ 気の合う同僚を       │
│ 回答する  │ 仲間を集める     │ 見つける             │
│ だけで    │                  │                     │
│ プロフ    │                  │                     │
│ ィール    │                  │                     │
│ が充実   │                  │                     │
└──────────┴──────────────────┴─────────────────────┘
```

## 4. 主要機能の詳細

### 4.1 AIインタビュー 💬

| 項目 | 内容 |
|---|---|
| **概要** | 毎日AIが出す質問（例：「最近現場で使っている技術は？」「週末の過ごし方は？」）に回答 |
| **体験** | チャット形式のUIで選択肢を選ぶ＋自由記述で気軽に回答 |
| **効果** | 回答内容をもとにプロフィール（AI紹介文・タグ）が自動的に充実する |
| **ポイント** | 社員が自らプロフィールを書く負担ゼロ。日々の回答で自然にアップデートされる |

### 4.2 グループ募集掲示板 📋

| 項目 | 内容 |
|---|---|
| **カテゴリ** | 🍽️ ごはん・飲み / 📚 勉強会・技術相談 / 🎉 イベント |
| **募集作成** | カテゴリ・タイトル・日程・エリア・オンライン可否を入力して投稿 |
| **レコメンド** | カテゴリとタグに基づき、おすすめメンバーを自動提案 |
| **参加アクション** | 「参加」「興味あり」「オンライン参加」の3段階で気軽にリアクション |
| **テーマサジェスト** | 「新宿でランチ行きませんか？」等の投稿テンプレートを提供 |

### 4.3 メンバーディスカバリー 👥

| 項目 | 内容 |
|---|---|
| **検索** | 名前・タグによるキーワード検索 |
| **タグフィルタ** | #React、#ラーメン、#登山 等のタグで絞り込み |
| **プロフィール表示** | AI紹介文・役職・エリア・入社日・タグを一覧表示 |
| **効果** | 「こんな人がいたんだ」という発見が日常的に起きる |

### 4.4 マイページ 👤

- プロフィール編集（名前・役職・エリア・タグ・紹介文）
- 自分の投稿履歴・AIインタビュー回答履歴の確認
- 初回ログイン時の初期設定ウィザード（アバター自動生成機能付き）

## 5. UI/UX のこだわり

| 観点 | 設計方針 |
|---|---|
| **モバイルファースト** | スマートフォンから片手で操作可能。通勤時間や昼休みに使える |
| **ボトムナビゲーション** | ホーム・掲示板・メンバー・マイページの4タブでシンプルに構成 |
| **低い参加ハードル** | 選択肢タップ＋一言で投稿・回答が完了。長文入力は不要 |
| **アバター自動生成** | 画像アップロードなしで、名前ベースのアイコンを自動作成 |
| **日本語ネイティブ** | 全UIを日本語で統一。エリア選択も国内拠点に最適化 |

## 6. 技術構成

### アーキテクチャ

```
[ユーザー端末 (Browser)]
        │
        ▼
  [Vercel CDN]  ── 静的ファイル配信
        │
        ▼
  [React SPA]   ── フロントエンド（React + TypeScript）
        │
        ▼
  [Supabase]    ── バックエンド
   ├── Auth      ── 認証（メール / OAuth）
   ├── Database  ── PostgreSQL + Row Level Security
   └── Realtime  ── リアルタイム更新（将来拡張）
```

### 技術スタック

| 技術 | 選定理由 |
|---|---|
| **React + TypeScript** | 型安全で保守性が高く、エンジニア採用が容易 |
| **Supabase** | BaaS により、バックエンド開発コストを大幅削減。PostgreSQL ベースで拡張性も確保 |
| **Vercel** | CI/CD が自動化され、Git push でデプロイ完了 |
| **shadcn/ui + Tailwind CSS** | 高品質なUIを短期間で構築可能 |

### セキュリティ

- **Row Level Security (RLS)**：全テーブルにアクセス制御ポリシーを適用
- **認証必須**：全機能にログインを要求（ルートガード実装済み）
- **本人のみ編集可**：プロフィール・投稿・参加情報はすべて本人のみ変更可能

## 7. 導入効果（期待される成果）

### 定量的効果

| 指標 | 現状（推定） | 導入後目標 | 測定方法 |
|---|---|---|---|
| 部門横断の交流回数 | 月1〜2回/人 | **月4回以上/人** | 掲示板の参加ログ |
| 新入社員のネットワーク形成 | 入社3ヶ月で5人程度 | **入社1ヶ月で10人以上** | メンバー間の接点数 |
| 社内勉強会の開催頻度 | 四半期に1回（有志） | **月2回以上** | 掲示板の勉強会カテゴリ投稿数 |
| AIインタビュー回答率 | - | **週3回以上（80%の社員）** | 回答ログ |

### 定性的効果

- **心理的安全性の向上**：業務外の接点が増えることで、業務上の相談もしやすくなる
- **組織の一体感**：部門・拠点を超えた「横のつながり」が生まれる
- **個人の可視化**：「AIからの紹介」で、業務だけでは見えない人柄が伝わる
- **自律的なコミュニティ形成**：社員自身が交流のきっかけを作れる仕組み

## 8. 導入計画

### フェーズ

| フェーズ | 期間 | 内容 |
|---|---|---|
| **Phase 1: パイロット導入** | 1ヶ月 | 特定部門（30〜50名）で試験運用。フィードバック収集 |
| **Phase 2: 改善 & 機能拡充** | 1ヶ月 | フィードバックを反映。通知機能・チャット連携等を追加 |
| **Phase 3: 全社展開** | 1ヶ月 | 全社員への展開。利用促進キャンペーンの実施 |
| **Phase 4: 定着 & 運用** | 継続 | 利用状況の分析。AI質問の更新サイクルの確立 |

### 運用体制

| 役割 | 担当 | 業務内容 |
|---|---|---|
| プロダクトオーナー | 人事部門 or 総務部門 | 方針決定、利用促進施策 |
| 開発・保守 | 社内エンジニア（1〜2名） | 機能追加、バグ修正 |
| AI質問作成 | HR / コミュニティ担当 | 週次でAI質問を更新 |
| コミュニティ運営 | 有志メンバー | 掲示板の活性化、イベント企画 |

## 11. リスクと対策

| リスク | 影響 | 対策 |
|---|---|---|
| 利用率が上がらない | 投資対効果が出ない | パイロット期間でのKPI設定、経営層からの利用推奨、ゲーミフィケーション要素の追加 |
| 不適切な投稿 | コンプライアンス問題 | 投稿カテゴリの限定（3種のみ）、通報機能の追加、管理者による監視体制 |
| セキュリティ懸念 | 情報漏洩 | Supabase RLS によるデータアクセス制御、認証必須化、HTTPS 通信 |
| 開発者の離任 | 保守困難 | React + TypeScript の標準的な構成で属人化を抑制、ドキュメント整備 |

## 12. まとめ

### Collabo-Connect が提供する価値

> **「社員の人柄を自然に可視化し、部門を超えた交流を日常に溶け込ませるプラットフォーム」**

| 従来のアプローチ | Collabo-Connect |
|---|---|
| 交流イベントは企画者頼み | 社員が自ら募集 → 自律的に交流が生まれる |
| プロフィールは自己紹介文を書く必要がある | AIインタビューに答えるだけで自動充実 |
| 誰が何に興味があるか分からない | タグ・AI紹介文でひと目で分かる |
| 交流は対面のみ | オンライン参加にも対応 |

### 次のステップ

1. **デモの実施** — 実際のアプリを操作いただき、体験を共有
2. **パイロット部門の選定** — 30〜50名規模での試験導入
3. **KPI の設定** — 交流回数・回答率等の目標値を合意
4. **Phase 1 開始** — 1ヶ月のパイロット運用
